When zero-coefficients don't the recursions terms

April 22, 2021
julia algorithms computation

Imagine you have a three-term recursion like this one

$$ g_{k+1} = A(l,k) g_{k} + B(l,k) g_{k-1},$$

where the \(g\)’s are items of the recursion, and \(A(l, k)\) and \(B(l, k)\) are coefficients who depends on some index \(l\). When we compute the recursion, the algorithm expects that the previous items are already calculated, but imagine this

g[1] = "some initial function"

for k in 1:N
	g[k + 1] = A(l,k) g[k] + B(l,k) g[k - 1]
end

where we don’t know the value of g[(k = 1) - 1] = g[0]. Assuming that B(l, k) = 0 for some \(l\) at \(k = 1\), it doesn’t matter we don’t know the value at g[1 - 1] right, or it matters?

Well, it depends on the programming language : Mathematica first looks for zero-coefficients and then ‘know’ that the recursion item won’t be used, so it doesn’t matter if their index maps to an uncalculating item. Julia, on the other side, ensures that the recursion item exists, no matter it goes to zero because their companion coefficients effectively vanished. This diametrical difference lead to quirks and perks: hard time to follow an error, the need to implement a conditional to evaluate the recursions properly, etc.
