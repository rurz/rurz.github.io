A seemly definition of the Kravchuk function on JuliaLang

April 15, 2021
julia algorithms computation

Intro

I’ve been used Julia on a daily basis for almost a year. Since the start one of my purposes was to get the Kravchuk function \(\Phi_{n}^{(j)}(q) \) correctly for at least 64 points, that is \(j = 32\). Since I came from Python and there the mpmath returns Hypergeometric functions without hesitation, I look for some like this in Julia. HypergeometricFunctions.jl is the (only?) package to bring the hyperthing \(_2F_1\). The Kravchuk function is defined as

$$(1)\qquad\qquad\Phi_{n}^{(j)}(q) = \frac{(-1)^{n}}{2^{j}}\sqrt{\binom{2j}{n}\binom{2j}{j+q}}k_{n}^{(2j)}(q),$$

where \(k_{n}^{(2j)}(q) = _2F_1(-n, -q, -2j, 2)\). This last function is the principal issue when we define (1), because the implementation on the package seems to have some specific issues.

1. Undefinedness at the corner when \(n = q = 2j\)

When we define (1), we use 2F1General2 who is a polyalgorithm on HypergeometricFunctions.jl, his purpose is to select the appropriate subroutine given the combination of parameters \({n, q}\). But when we evaluate a matrix of (1) in the ranges of the parameters, at the corner a NaN appears. The first solution was to create a conditional, since because this point is the reflection of the evaluation at \(n = q = 0\).

This last permits us to treat calculations fairly enough to reach \(j = 20\), above this value, rises another issue.

2. Bigger \(j\) gives bigger problems

When \(j > 20\) we experience inconsistencies, overflows and dumped cores. This seems to be caused by the parameters \(n\) and \(q\), but principally \(n\). I think 2F1General2 don’t play very well with the combination of values. So, we faced a BigFloat problem, easily solved when we identify what’s the matter. So we do 2F1General2(BigFloat(-n), BigFloat(-q), -2j, 2). This solves the problem, well, partially solves the problem, because we can reach \(j = 128\) without problems, but the good behavior falls quickly above this value.

And, to avoid problems with bigger \(n\) we can use the mirror symmetry of (1), this is, when \(0 \le n \le j\) we use (1), but for \(j < n \le 2j\) we implement

$$\Phi_{2j-n}^{(2j)}(q) = (-1)^{q}\Phi_{n}^{(2j)}(q),$$

who also solves the problem at section 1, avoiding to use a conditional over the hypergeometric.

This quantity of points correctly obtained are fairly enough for the moment, because the purpose for obtain Kravchuk functions with at least \(j = 128\), is to use it in the unitary rotation of polychromatic pixelated images1 . The implementation is available at the GitHub repo -> https://github.com/rurz/VisitingICF2021

  1. Urzúa, A. R., & Wolf, K. B. (2016). Unitary rotation and gyration of pixelated images on rectangular screens. Journal of the Optical Society of America A, 33(4), 642. https://doi.org/10.1364/josaa.33.000642 ↩︎
